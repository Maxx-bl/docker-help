TP 02

Asterisk (VoIP)

SIP · Linphone · Dockerfile avancé

▼
Nettoyage initial
docker rm --force $(docker ps -aq)Supprimer tous les conteneurs
docker pull debianou docker load -i debian.tar.gz
Conteneur Asterisk
docker run -ti --name asterisk-dev --network host debianRéseau hôte obligatoire pour SIP
apt update && apt install -y asteriskDans le conteneur
docker cp asterisk-dev:/etc/asterisk/sip.conf ./Récupérer le fichier de config
docker cp asterisk-dev:/etc/asterisk/extensions.conf ./
docker cp sip.conf asterisk-dev:/etc/asterisk/sip.confRemettre le fichier modifié
asterisk -vvvvvvcLancer Asterisk (dans le conteneur)
sip.conf → ajouter [alice] et [bob] avec type=friend, secret, context=infoS4
extensions.conf → [infoS4] exten => 100,1,Dial(SIP/alice) …
Conteneur Linphone
docker run -ti --name linphone-dev --network host --device /dev/snd debianAccès carte son
apt install -y linphone-cliDans le conteneur
mkdir -p /root/.local/share/linphone
Commandes Linphone (dans linphone-dev) :
linphonec → soundcard list → soundcard use 2
register sip:alice@IP_serveur IP_serveur mdp-alice
call 101  |  answer  |  terminate
Dockerfile Asterisk (volume)
FROM debian
RUN apt update && apt install -y asterisk
CMD ["asterisk", "-vvvvvvc"] Dockerfile minimal
docker build -t mon-asterisk .Construire l'image
docker run --network host -v ./conf:/etc/asterisk mon-asteriskMonter les fichiers de config
TP 03

Nagios (Monitoring)

Apache · SNMP · Surveillance réseau

▼
Nettoyage + lancement
docker swarm leave --forceQuitter le swarm précédent
docker rm --force $(docker ps -aq)
docker run -ti --name nagios -p 8080:80 debianConteneur Nagios, port 80→8080
apt update && apt install -y nagios4 iproute2 procps nanoDans le conteneur
Configuration Apache + Nagios
a2enmod cgi rewriteActiver modules Apache
htdigest /etc/nagios4/htdigest.users "Nagios Access" nagiosadminCréer le compte admin
apachectl startDémarrer Apache
nagios4 /etc/nagios4/nagios.cfgDémarrer Nagios (noter le PID)
nagios4-cgi.conf : commenter Require ip, remplacer <Files "cmd.cgi"> par <FilesMatch "">, activer Require valid-user
cgi.cfg : use_authentication=1
Recharger la configuration
kill -HUP <PID_nagios>Recharger sans redémarrer
docker exec -ti nagios bashOuvrir un 2ème terminal dans le conteneur
SNMP — Surveillance imprimantes
snmpwalk -v2c -c public IMPI118 -m ALL > oids_numeriquesRécupérer tous les OIDs
apt install -y snmp-mibs-downloader && download-mibsInstaller les bases MIB
snmpwalk -v2c -c public IMPI118 -m ALL > oids_symboliquesOIDs avec noms symboliques
/usr/lib/nagios/plugins/check_snmp -H IMPI118 -C public -o OID_toner -l "Toner level" -u " sheets" -w 10000: -c 1000:Tester le plugin SNMP
Fichier /etc/nagios4/objects/printer.cfg : définir hôtes IMPI118/IMPI120 + services PING + check_snmp
check_command check_snmp!-C public -o OID_toner_courant -l "Toner level" -u " sheets" -w 10000: -c 1000:
TP 04

Docker Swarm

Cluster · Services · Répartition de charge · Config

▼
Initialisation du cluster
docker swarm leave --forceQuitter l'ancien swarm
docker rm --force $(docker ps -aq)Nettoyer les conteneurs
docker swarm init --advertise-addr IP_eno1_managerCréer le swarm (sur le Manager)
docker swarm join --token TOKEN IP_eno1_managerRejoindre le swarm (sur chaque Worker)
docker node lsVérifier les nœuds du cluster
Gestion des services
docker service create --name web --replicas N nginxCréer un service répliqué
docker service lsLister les services
docker service ps webVoir les instances du service
docker psInstances locales (sur chaque nœud)
Tolérance aux pannes
docker swarm leaveWorker quitte le swarm
docker node lsSurveiller l'état des nœuds
docker rm --force <ID_conteneur>Supprimer une instance (Docker en recrée une)
Mise à jour du service (publication port · montage)
docker service update --publish-add 8080:80 webPublier port 80 → 8080 sur tous les nœuds
docker service update --mount-add type=bind,src=/media/tp/S4/Virtualisation/TP3,dst=/usr/share/nginx/html webMonter un répertoire NFS partagé
docker service update --mount-rm /usr/share/nginx/html webDémonter un volume
Test de répartition de charge : wget --no-proxy -nv -O- http://IP_instance/me.html
Recharger plusieurs fois → les requêtes sont distribuées entre les nœuds.
Docker Config (partage de fichiers < 500 ko)
docker config create life /media/tp/S4/Virtualisation/files/game_of_life.htmlCréer un config
docker service update --config-add source=life,target=/usr/share/nginx/html/vie.html webAjouter le config au service
Inspecter · Copier (sur les instances)
docker inspect <ID_conteneur>Adresse IP d'une instance
docker cp me.html <ID_conteneur>:/usr/share/nginx/html/me.htmlCopier fichier dans une instance
